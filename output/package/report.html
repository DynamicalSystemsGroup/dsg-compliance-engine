How to read this report

A few terms recur throughout. In plain language:

Control
One specific security requirement from NIST SP 800-171 — for example, "lock the screen after inactivity" or "keep audit logs." There are 110 in all; this contract requires a defined subset.
SPRS score
The single number the Department of Defense uses to summarize a self-assessment. It starts at 110 and subtracts a weight (1, 3, or 5) for each control not met. A higher number is better; 110 is a perfect score.
Met / not met
A control is recorded as met only when its evidence passed the check for it and a named, authorized person signed off. Anything short of both is not met.
Machine-verified vs. human-attested
Some controls can be proven by software re-reading system configuration (machine-verified). Others — training, policy, personnel — can only be confirmed by a person reviewing documents (human-attested). Both are legitimate; they differ in how an assessor re-checks them.
Attested, machine-backed
A middle ground for the policy controls: a person signs off, but the document they are vouching for is machine-recorded — the engine resolved the actual file, fingerprinted it (SHA-256), captured the git commit that produced it, and holds a signed receipt of its upload. So the human judgment is anchored to a specific, traceable version of a real document rather than a bare "trust me."
Override
A control marked met even though its automated check failed. This is allowed, but it must carry a written justification and supporting evidence, and it is flagged for review.
Signed / verifiable
The whole package is sealed with a digital signature and every file is fingerprinted. If anyone changes a single character after signing, re-verification fails. That is what lets a reader trust this report without trusting whoever produced it.

The score

The SPRS score is computed over the 110 controls this contract requires. Each control carries a weight of 1, 3, or 5 reflecting how much it matters to protecting controlled information; the score is 110 minus the weights of the controls not met. A valid submission additionally means no high-weight control was left open on a plan of action where the rules forbid it.
